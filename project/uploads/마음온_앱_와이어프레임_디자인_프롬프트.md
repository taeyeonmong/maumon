# 마음온 시니어 대화앱 — 와이어프레임 제작 요청

## 역할 및 목표

당신은 시니어 복지 앱 전문 UX 디자이너입니다.
아래 기능명세서를 바탕으로 **마음온 시니어 대화앱**의 iPad 와이어프레임을 제작해 주세요.

---

## 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 마음온 (Maeum-ON) |
| 목적 | 독거노인 고독사·고립 예방 AI 컴패니언 |
| 디바이스 | iPad (단일 기기, MDM/Guided Access 키오스크 모드) |

## 핵심 설계 원칙

### 홈 화면 구조
- **카드 구조** (탭 구조 아님) — 홈 한 화면에서 모든 기능 카드로 접근
- 상단 고정: AI 안부콜 상태 카드 (다음 콜 시각, 오늘 대화 횟수)
- 2열 그리드: 상담예약 / 채팅상담 / 오늘의 활동 / 대화기록
- 하단: 공지사항 (한 줄 배너)

### 시니어 UX 필수 조건
- 터치 영역 최소 **48px 이상** (권장 60px+)
- 글씨 크기 최소 **18px** 이상 (중요 텍스트 24px+)
- 고대비 색상 (메인 컬러: `#1a5fa8` 파랑 계열)
- 화면당 **주요 액션 1개** 원칙
- 버튼은 텍스트 + 아이콘 병기
- 경고/안내 문구는 명령형 대신 **권유형·대화체** 사용

### 위험도 색상 체계 (일관 적용)
- Lv.0~1 (평상·관찰): 파랑 `#1a5fa8`
- Lv.2 (주의): 주황 `#e07b2a`
- Lv.3 (위험): 빨강 `#dc2626`
- Lv.4 (긴급): 진빨강 배경 전체 전환

## 와이어프레임 제작 대상 화면

### P0 — 안부콜 핵심 플로우 (7/22 통합테스트 필수)

#### S01. 홈 화면 (평상시 · Lv.0)
- 앱 헤더: 마음온 로고 + 알림벨 + 설정 아이콘
- AI 상태 카드 (풀 너비): AI 아바타 + 이름 + "다음 안부콜 XX분 후" 카운트다운
- 2×2 기능 카드: 상담예약 / 채팅상담 / 오늘의 활동 / 대화기록
- 하단: 공지사항 한 줄 배너

#### S02. 안부콜 진입 — 전환 화면
- 전체 화면 전환 (홈 완전 교체)
- 배경: 딥 블루 전체
- 중앙: AI 아바타 + "홍길동님, 점심 안부 드릴게요" 텍스트
- 3초 자동 시작 카운트다운 진행바
- 하단: "바로 시작할게요" 버튼 (보조)

#### S03. 대화 진행 — AI 발화 중
- 헤더: 콜 종류 + 우측 "종료" 버튼 (소형, 빨강)
- 중앙: AI 아바타 (발화 중 애니메이션 링)
- STT 자막 말풍선
- 하단: 음성 파형 인디케이터

#### S04. 대화 진행 — 사용자 발화 중
- S03 동일 레이아웃
- AI 아바타 → "듣는 중" 상태 (시각적 변화)
- 하단: 사용자 음성 파형 (초록) + 마이크 아이콘 활성
- 30초 침묵 타이머 표시

#### S05. 감정 확인
- 헤더: "오늘 기분이 어땠나요?"
- 4개 버튼 (풀 너비, 세로 리스트, 각 60px+ 높이):
  - 매우 좋아요 / 좋아요 / 보통이에요 / 힘들어요 (각 아이콘 + 텍스트)
  - "힘들어요"는 빨강 테두리로 강조

#### S06. 대화 완료
- 배경: 연초록
- 중앙: 체크 아이콘 + "수고하셨어요, 홍길동님!"
- 다음 안부콜 정보 카드
- 5초 자동 홈 복귀 / "홈으로" 버튼

#### S07. 상담 연결 제안 (Lv.2)
- 배경: 연주황
- 상담사 아이콘 + "상담사 선생님과 이야기 나눠보시겠어요?"
- 버튼 2개: "채팅으로 바로 연결" (주) / "예약하기" (보조)
- "괜찮아요 (건너뛰기)" 텍스트 링크

#### S08. 긴급 연결 화면 (Lv.3)
- 배경: 전체 빨강
- 헤더 배너: "위기 감지"
- 중앙: 전화 아이콘 + "걱정이 되어 선생님께 연락 드릴게요"
- 연결 중 스피너 + "담당 선생님께 알림을 보냈습니다" 완료 메시지
- 사용자 조작 불필요 (자동 처리 화면)

---

### P1 — 보조 기능

#### S09. 상담 예약 확인
- 예약 카드: 상담사 이름 + 날짜·시간 + 유형 아이콘
- D-day 카운트다운
- "입장하기" 버튼 (10분 전 활성화)
- 예약 없음 빈 상태 화면

#### S10. 채팅 상담
- 헤더: 상담사 이름 + 온라인 상태 표시
- 채팅 UI: 상담사(좌) / 본인(우) 말풍선
- 하단: 음성 입력 버튼 (주) + 텍스트 입력 (보조)

#### S11. 오늘의 활동 (콘텐츠)
- 오늘의 추천 카드 (풀 너비)
- 콘텐츠 목록 (최대 3개): 추억 퀴즈 / 회상 대화 / 생활정보
- 완료 항목 체크 표시

#### S12. 대화 기록 조회
- 날짜별 카드 리스트 (최근 7일)
- 각 카드: 날짜 + 트리거 유형 + 요약 한 줄 + 감정 표시
- "음성으로 읽어주기" 버튼

#### S13. 음성 설정
- 음량: 슬라이더 (현재값 숫자 표시)
- 말 속도: 느리게 / 보통 / 빠르게 3버튼 선택
- 음색: 카드 선택 (2~4개)
- 각 항목 옆 "미리 듣기" 버튼
- 하단 고정 "저장" 버튼

#### S14. 공지사항
- 공지 목록 (최신순, 읽지 않음 파랑 점)
- 상세: 큰 글씨 본문 + "음성으로 읽어주기" 버튼

### P2

#### S15. 오프라인 안내
- 와이파이 끊김 아이콘 (크게)
- "인터넷 연결이 끊겼어요"
- 담당 복지사 전화번호 (탭 → 전화 연결)
- 재연결 시도 스피너

## 출력 형식 요청

- **iPad 세로 방향** 기준 (768×1024pt)
- **그레이스케일 + 포인트 컬러** (위험도 컬러·메인 블루만 적용)
- 각 화면에 **화면 번호 + 화면명** 레이블
- 주요 UI 요소 **어노테이션** (번호 + 설명)
- 화면 간 **연결 관계** 표시 (화살표 + 전환 조건)
- P0 (S01~S08) 먼저 제작 → 피드백 후 P1·P2 진행

## 확정 설계 결정 사항

| 항목 | 결정 내용 |
|------|-----------|
| 홈 구조 | 카드 구조 확정 (탭 구조 기각) |
| 안부콜 구현 | iPad 키오스크 모드 — 스케줄러가 화면 전환 트리거 |

| SOS 버튼 | 항상 우측 상단 고정 (Lv.4 동작 미결) |
| 로그인 | 복지사가 초기 세팅 (시니어 직접 입력 없음) |
